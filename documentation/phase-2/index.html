Part 2: Ticket System

We will be using the commands of the Ticket tool bot, because the commands are almost always the same in all ticket bots.

As with all bots, don't forget to check the bot's prefix.

To make a new ticket, type $new. Some servers may use $ticket.

To add a user to a ticket, type $add @user #channel. Example: $add @user #ticket-1048

To remove a user from a ticket, type $remove @user #channel. Example: $remove @user #ticket-1048

To close a ticket, type $close in the ticket channel.

To save the transcript of a ticket (a log of all messages sent inside a ticket) type $transcript #channel

To delete an already closed ticket, type $delete in the ticket channel.

IMPORTANT TIPS:

Tagging: You should never tag staff directly in a ticket channel, you should always tag staff in a staff channel.

Here's how you should do it: @user #channel (do you need help or do they need to take over the ticket): (situation)

Example: @user #ticket-2049 please take over: partner request

Who has the response responsibility inside a ticket?

If the staff doesn't respond to a ticket in a chosen (by the higher-ups) amount of time, you should tag them in a staff channel to let them know.

Never argue with a fellow staff member inside a ticket; always discuss/argue in a staff channel or in DMs.

You can always give tips or tell them your opinion about the situation, but you should avoid doing it inside the ticket.

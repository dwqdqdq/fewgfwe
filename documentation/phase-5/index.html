Discord Moderation Training - Phase 5

{Server protection}

A server should always have some form of security to protect it from raids and nuke attacks. In this phase, we will talk about using Wick Bot and Xenon Bot to protect your server.

Wick Bot

Wick is a moderation, security, and utility bot made to help the server owners protect their server from raids, mass spam attacks, and rogue staff members.

Lockdown

Lockdown commands are very useful as they disable typing in a particular channel or the whole server. It can also blind the channels which makes the channels invisible to the users. This can make stopping raiders easier.

We will now look at the lockdown commands.

w!lock #channel-name

This command will lock the mentioned channel which will disable typing in that channel.

w!serverlock

This command will lock the whole server which will disable typing in every channel of that server.

w!serverlock -blind

This command will lock all the channels of the server and as well as make them invisible to the users.

w!lock ?t [ban/kick]

This command will ban or kick every new member who joins after this command is run.

Quarantine

The quarantine role is given to a user when a trusted admin, extra owner, or anyone with the Quarantine permit permission runs a quarantine command on them. It can also be given when a user with dangerous permission triggers an automod filter like mass mentions, or when they trigger your anti-nuke. When one of these happens, the user is stripped of any/all power, and roles, and is given this role, until someone unquarantines them.

We will look into the commands now.

w!q @user ?r (reason)

This command quarantines the mentioned user and the user is stripped of powers if any are present as well as their roles.

w!uq @user ?r (reason)

Unquarantines the user and the user gets back their roles.

B. Xenon Bot

The Xenon bot is used for readymade server templates and server backup in many servers. It copies your server's channels and roles and stores it in a backup. When a copy of the server is needed, the backup is copied to a new server. This can save a lot of time in case something happens to the server.

Let us look at the commands of this bot.

/backup create

Creates a backup of the server in which the command is run.

/backup list

/backup intervalon (time)

Sets the interval of backups for example 24hr, 2days, etc.

/backup load (backup id)

Use Xenon and run /backup list to find out the backup ID of the desired server and use /backup load (backup ID) to load the backup.
